    For each repository:
    - Use `gh repo create pebaum/[project-name] --public`
    - Do NOT initialize with README (content will be migrated in Phase 2)
    - Use exact project folder names from monorepo

    Note: Skip if repository already exists (gh will warn, continue to next).
  </action>
  <verify>

  </verify>
  <done>
    All 10 repositories exist on GitHub under pebaum account and are visible via gh CLI.
  </done>
</task>

<task type="auto">
  <name>Task 2: Enable GitHub Pages for all repositories</name>
  <files>None (configures remote GitHub settings)</files>
  <action>
    Enable GitHub Pages for each of the 10 repositories using gh API:

    For each repository, run:
    ```bash
    gh api repos/pebaum/[project-name]/pages -X POST -f source[branch]=master -f source[path]=/
    ```

    Repositories:
    - 4-track-tape-looper
    - dnd-dm-screen
    - drift
    - dualshock-synth
    - forward-boardgame
    - gelatinous-cube-puzzle
    - granular-ambient
    - generative-web-art
    - nova4
    - textscape

    Note: If Pages already enabled (409 conflict), that's acceptable - continue to next repository.

    Wait 30-60 seconds after enabling Pages for GitHub to begin building sites.
  </action>
  <verify>
    For each repository, verify Pages is enabled:
    ```bash
    gh api repos/pebaum/[project-name]/pages
    ```

    Should return JSON with "status": "built" or "status": "building" and "html_url": "https://pebaum.github.io/[project-name]/"
  </verify>
  <done>
    GitHub Pages is enabled for all 10 repositories and each repository reports a valid html_url endpoint.
  </done>
</task>
